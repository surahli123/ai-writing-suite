# v1 Test Plan (2026-06-07)

Owner decision: **Phase 1 (regression gate + CI) first, then Phase 2 (quality eval).**
Status: PLAN — not yet built. Install/load smoke (option a) is already DONE this session.

Context: v1's existing tests are all *mechanical* (the Python AI-tell detector + fixtures +
KB smoke). The product's actual job — `comms-polish` removing AI tells while preserving meaning
and voice — is **unmeasured**: `evals/fixtures/rubric.md` defines the LLM-judge dimensions but no
model is wired in (it emits SKIPPED). Detector ≠ product (the detector deliberately misses the
subtle cases — that's the 38% calibration).

---

## Phase 1 — Regression gate + CI  (do first; cheap insurance)

**Goal:** one command + CI that runs every existing check, so future edits (especially
self-improvement rule additions) can't silently regress. Makes the "eval-gated self-improvement"
claim real.

**Build:**
1. `evals/run_all.sh` (or a Makefile target) that runs, in order, failing on any nonzero:
   - `python3 -m unittest discover -p 'test_*.py'`  (23 tests)
   - `python3 smoke_test.py`                          (KB smoke, 3 cases)
   - `python3 -m fixtures.run_fixtures`               (deterministic half + 30-40% calibration assert)
2. `.github/workflows/ci.yml` — runs `run_all.sh` on push + PR. **Stdlib-only**, so: `ubuntu-latest`,
   `actions/checkout`, `setup-python`, one run step. No deps to install.
3. (optional) codify the code-review doc's adversarial detector inputs (empty / whitespace / emoji /
   CJK / 35k-words) as explicit `test_detector` cases so crash-proofing is guarded, not just recorded.

**Effort:** ~1 hr. **Done =** green CI badge; a deliberately-broken fixture turns CI red.

---

## Phase 2 — Quality eval  (the real gap; do after Phase 1)

**Goal:** measure whether `comms-polish` actually produces good output — the core promise.

**Design (DS-flavored, because the judge is itself a measurement instrument):**
1. **Eval set:** assemble ~15-25 real before/after items (real AI-shaped drafts across the 4 genres),
   beyond the 8 synthetic fixtures. Label the "after" quality by hand on the rubric dimensions.
2. **Wire the judge:** turn `run_fixtures.py --judge` from SKIPPED into real verdicts — a model scores
   each rubric dimension (meaning_preserved, tells_removed, **no_fabrication** [highest stakes],
   voice_kept, specificity_added, genre_fit).
3. **Calibrate the judge BEFORE trusting it** (broken-ruler guard): on the hand-labeled subset,
   measure judge-vs-human agreement; check for the usual LLM-judge biases (position, verbosity,
   self-preference). Only report polish-quality numbers once the judge tracks human ratings.
4. **Metric:** per-dimension pass rate + before→after tell-density delta (detector as the cheap
   secondary signal, judge as the relevance label). Set a release threshold.

**Open decisions for Phase 2 (defer until we start it):**
- Which model as judge? (cross-model to avoid self-preference, e.g. judge with a different family
  than the one doing the polish.)
- Eval-set source: owner's real drafts vs synthetic-but-realistic vs a public AI-text corpus.
- Human-label sample size for calibration.

---

## Not in the core sequence
- **(d) Detector accuracy on a labeled set** — deprioritized for v1: the detector is a secondary
  signal; the LLM does the real polish. Revisit if the detector becomes a hard gate.
- **(e) Dogfood** — complementary confidence check; also the only thing that exercises `voice-onboard`
  (0 tests). Good as a "does this actually help me" gut-check, weak as a regression guard.
